Executive Summary

The Lumion AI Assistant's school-tier system prompt has been completely rewritten from a minimal 14-line instruction set to a 240+ line, research-backed prompt architecture. This work was informed by a comprehensive research document (unified-guide.md) that synthesizes production guidance from seven authoritative sources.

The new prompt introduces analytical rigor (statistical trap detection, data grounding prohibitions), domain locking (CRM-only scope with warm off-topic handling), security guardrails (multi-tenant data isolation), and structured data presentation (layered insight delivery). It also upgrades the school tier from GPT-5-mini to GPT-5.2 for stronger tool-calling performance.

Supporting infrastructure changes include an N+1 query fix in the AgentOrchestrator, unresolved placeholder detection, timezone safety, and tier-aware model routing.

Supporting Technical Improvements

N+1 Query Fix

AgentOrchestrator now uses loadMissing('roles') to prevent redundant database queries when resolving user roles for prompt interpolation.

Placeholder Detection

After prompt interpolation, a regex scan detects any unresolved {{placeholders}} and logs a warning — catching configuration mismatches before they reach users.

Timezone Safety

New safeTimezone() helper validates IANA timezone strings before use, falling back to UTC on invalid input. Prevents exceptions from corrupted user preferences.

Tier-Aware Model Routing

Model selection is now tier-aware: school conversations use GPT-5.2 (strongest tool calling), student conversations use GPT-5-mini (highest groundedness, lowest cost).
